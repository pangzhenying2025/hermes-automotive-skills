# L3 Fusa Sotif — China Standard

# L3 有条件自动驾驶系统 功能安全和预期功能安全技术要求

## 标准信息

| 属性 | 值 |
|------|-----|
| **标准编号** | 待定（GB/T） |
| **名称** | 智能网联汽车 有条件自动驾驶系统 功能安全和预期功能安全技术要求 |
| **状态** | 工作组草案（2025.01.18 正式版） |
| **推荐等级** | P2-推荐入选 |
| **特殊价值** | 中国独有标准，全球首个将FuSa+SOTIF联合要求系统化的标准 |
| **总页数** | 132页 |

## 标准定位

```
L3 FuSa+SOTIF联合标准的独特性
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ISO 26262 (FuSa) ← 独立标准
  ISO 21448 (SOTIF) ← 独立标准
  → 两者如何在L3系统中协同？缺少指导

中国创新:
  本标准首次将FuSa+SOTIF联合要求系统化
  ├── 统一的安全目标体系
  ├── 联合的风险分析方法

  └── 综合的安全论证框架

引用基础标准:
  GB/T 34590-2022 (功能安全)
  GB/T 43267-2023 (预期功能安全)
  GB/T 40429-2021 (驾驶自动化分级)
  GB/T 44721-2024 (自动驾驶系统通用技术要求)
```

### 残余风险接受准则

```
残余风险接受准则
├── 碰撞频率：不超过 10⁻⁴/小时
├── SOFIL等级：最低 4E 级
├── 与人类驾驶员安全水平对比
│   ├── 不低于合格谨慎驾驶员水平
│   └── 基于自然驾驶数据的统计论证
└── 持续安全监控义务
```

### MRM（最小风险操作）安全要求

```
MRM安全要求
├── 触发条件
│   ├── 系统故障检测
│   ├── ODD退出检测
│   ├── 驾驶员持续无响应
│   └── 传感器严重退化
├── 执行要求
│   ├── MRM启动延迟 < 500ms
│   ├── 减速度限制（舒适性兼顾）
│   ├── 优先选择安全停车位置
│   └── 全程保持碰撞避免
├── 安全停车
│   ├── 开启双闪警示
│   ├── 驻车制动激活
│   ├── 解锁车门
│   └── 触发eCall（如严重情况）
└── 验证要求
    ├── 仿真测试覆盖所有MRM场景
    ├── 场地测试验证物理执行
    └── 道路测试验证真实工况
```
